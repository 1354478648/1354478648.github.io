Git的学习笔记

发表于 2023-01-31 更新于 2023-10-10 分类于 后端 阅读次数：
本文字数： 4.2k 阅读时长 ≈ 8 分钟

Git是一个免费的、开源的分布式版本控制系统，可以快速高效地处理从小型到大型的各种项目

  • **版本控制：**主要功能是追踪文件的变更
  • 集中式版本控制系统：例如CVS、SVN等等，都有一个单一的集中管理的服务器，保存所有文件的修订版本。协调工作的人们都通过客户端连接到这台服务器，提交更新或者取出最新的文件
  • 分布式版本控制系统：客户端把代码仓库完整地镜像下来（本地库）。因为每个客户端的每一次文件提取操作，实际上都是一次对整个文件仓库的完整备份，这样任何一处协调工作用的文件发生故障，事后都可以用其他客户端的本地仓库进行恢复
  • 工作机制：
    工作区 ———git add———> 暂存区 ———git commit———> 本地库 ———git push———>远程库
  1. 工作区：保存本地代码的磁盘位置
  2. 暂存区：临时存储
  3. 本地库：生成一个历史版本
  4. 远程库：代码托管中心
  • 代码托管中心
    代码托管中心是基于网络服务器的远程代码仓库，称为远程库
    局域网：GitLab；互联网：GitHub（外网），Gitee 码云（国内）

分支

在版本控制过程中，同时推进多个任务。为每个任务，我们就可以创建每个任务的单独分支。使用分支意味着程序员可以把自己的工作从开发主线上分离开来，开发自己分支的时候，不会影响主线分支的运行

  • **分支的好处：**同时并行推进多个功能开发，提高开发效率；各个分支在开发过程中，如果某一个分支开发失败，不会对其他分支有任何影响，失败的分支删除重新开始即可

代码冲突问题

合并分支时，两个分支在同一个文件的同一个位置有两套完全不同的修改。Git无法替我们决定使用哪一个，必须人为修改并决定新代码内容
**注意：**修改完成后，提交本地库命令git commit -m "日志信息"是不能带文件名的

  • 切换分支的本质是移动HEAD指针；HEAD指针指向BRANCH指针，BRANCH指针里面保存着所有历史版本

团队内协作

  • 添加团队成员

跨团队协作

  • 复制远程库：团队外成员点击右上角Fork --> 远程库资源被复制到团队外成员的远程库中 --> 团队外成员可以在线编辑代码，也可以clone到自己本地库
  • 发送修改好的代码：Pull requests --> New pull request --> Create pull request --> 编辑标题或留言
  • 团队内成员审核代码：查看Pull requests --> 审核通过后Merge pull request合并申请 --> 点击“确定合并”

以Maven项目为例添加Git追踪

  • 提交暂存区
    选中一个项目 --> 点击上方菜单栏VCS-Create Git Repository --> 会自动选中当前项目文件夹(没有自动选中，可以手动选中) --> 打开项目文件夹会发现已经创建.git文件夹，且IDEA里POM.XML变成红色，说明POM.XML还未添加到暂存区 --> 右键POM.XML-Git-Add --> POM.XML变绿说明成功添加到暂存区
    注：如果写了多个代码文件，可以直接右键项目根目录，提交到暂存区
  • 提交本地库
    右键项目根目录-Git-Commit Directory --> 输入commit message内容(就是commit命令中-m里填写的内容) --> 点击commit

切换版本

以提交了三个版本的代码为例 --> 左下角点击Git就可以看到三个版本 --> 右键某个版本-checkout Revision 版本号，即可切换版本

创建分支&切换分支

右下角点击版本号(或分支名) --> New Branch --> 起分支名

合并分支

  • 正常合并
    来到master分支 --> 右下角点击版本号(或分支名) --> 选中hot-fix分支 --> merge ‘hot-fix’ into ‘master’
  • 代码冲突
    merge代码后出现警告弹窗 --> 点击merge --> 左右两边是修改过的代码，中间是没修改过的代码

设置GitHub账号

进入GitHub --> 点击头像-设置-Developer Settings-Personal access tokens --> 权限全部选中 --> 复制token(注意：刷新后就消失了) --> 回到IDEA -->点击Files-Settings-Version Control-GitHub --> 使用token登录，粘贴token

推送到远程库

上方菜单栏Git --> Share Project on Github --> 起别名
如果发现GitHub创建了远程库，但没有代码文件，可以尝试右下角选中分支-点击Push
使用SSH免密登录：右下角点击分支-点击Push --> 点击项目名 --> 点击Define Remote --> 起别名，粘贴SSH链接 --> 再次点击项目名，选中设置的别名 --> 点击Push

拉取远程库到本地库

建议：在动手修改本地代码之前，先检查远程库最新的代码和本地老的代码的区别，如果本地库的代码已经落后，切记前pull远程库的代码，将本地代码更新到最新，再提交推送
上方菜单栏Git-Pull --> 在弹窗中设置需要拉取的分支
注意：如果自动合并失败，需要手动解决代码冲突问题

克隆远程库

在IDEA初始化页面选择Get From Version Control --> 在Repository URL中粘贴HTTPS或SSH链接，设置克隆的项目路径
